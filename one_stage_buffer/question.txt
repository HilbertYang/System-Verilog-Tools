题目：单级缓冲握手模块

设计一个 1-stage buffer，接口如下：

输入：

clk
rst_n
in_valid
in_data[31:0]
out_ready

输出：

in_ready
out_valid
out_data[31:0]

要求：

模块位于上游和下游之间，用于缓存 1 个 32-bit 数据。
只有 in_valid && in_ready 时，才能从上游接收新数据。
只有 out_valid && out_ready 时，才表示下游成功取走数据。
当 buffer 里有数据时，out_valid=1；当 buffer 空时，out_valid=0。
当 buffer 满时，不能继续接收新数据。
写出 in_ready 的组合逻辑表达式，并解释原因。

面试常见追问：

这个模块和 FIFO 有什么区别？
为什么 valid 不能依赖 ready 来产生？
如果要求支持“同拍出旧数据、同拍进新数据”，你逻辑怎么写？

English Version:

Problem: Single-Stage Buffer Handshake Module

Design a 1-stage buffer with the following interface:

Inputs:

clk
rst_n
in_valid
in_data[31:0]
out_ready

Outputs:

in_ready
out_valid
out_data[31:0]

Requirements:

The module sits between an upstream producer and a downstream consumer, and is used to buffer one 32-bit data word.
The module can only accept new data from upstream when in_valid && in_ready is true.
Data is considered successfully consumed by downstream only when out_valid && out_ready is true.
When the buffer contains data, out_valid must be 1; when the buffer is empty, out_valid must be 0.
When the buffer is full, it must not accept additional new data.
Write the combinational logic expression for in_ready, and explain why.

Common Interview Follow-up Questions:

What is the difference between this module and a FIFO?
Why must valid not be generated based on ready?
If the design is required to support "output old data and accept new data in the same cycle," how would you write the logic?
